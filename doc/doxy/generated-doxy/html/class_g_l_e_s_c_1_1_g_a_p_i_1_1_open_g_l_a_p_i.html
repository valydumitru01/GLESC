Configures the layout of the vertex data.

This method sets up how OpenGL interprets the vertex buffer data. It is used to specify the location and data format of the array of generic vertex attributes.

Parameters
indexThe index of the vertex attribute to be modified.
countThe number of components per generic vertex attribute. Typically, this is 1, 2, 3, or 4.
typeThe data type of each component in the array. For example, GL_FLOAT, GL_INT, etc.
isNormalizedSpecifies whether fixed-point data values should be normalized or converted directly as fixed-point values when they are accessed.
strideThe byte offset between consecutive vertex attributes. If the attributes are tightly packed, this value is 0.
offsetThe offset of the first component of the first vertex attribute in the buffer. This is usually a byte offset.
